# OCC-RAG Paper

Исследование по OCC-RAG Reader и интеграции в систему КРЕПОСТЬ.

## OCC-RAG: компактный контекстно-верный reader

OCC-RAG (Output-Constrained Context-faithful RAG) — это компактная модель-reader размером **0.6B или 1.7B параметров**, специально разработанная для задач RAG. Главное свойство: модель является **context-faithful by design** — она не галлюцинирует за пределами предоставленного контекста.

## Проблема, которую решает OCC-RAG

Стандартные LLM (включая Qwen3.6-27B) при ответе на вопрос по контексту могут «додумывать» информацию из своих параметрических знаний, что создаёт риск галлюцинаций. OCC-RAG архитектурно ограничена: она может генерировать ответы только на основе предоставленного контекста, полностью исключая параметрические галлюцинации.

## Экономия ресурсов

Использование OCC-RAG вместо Qwen3.6-27B для задач чтения (извлечение информации из контекста) даёт экономию **17GB RAM**:
- Qwen3.6-27B (Q4_K_M): ~16GB
- OCC-RAG (1.7B): ~1.2GB (при INT4 квантизации ещё меньше)

Это освобождает ресурсы Mac Studio для параллельной работы Guard, эмбеддингов и других компонентов.

## Patch v1.1: интеграция с retriever_protocol

В версии Patch v1.1 OCC-RAG интегрирован с **retriever_protocol** — унифицированным протоколом взаимодействия с различными поисковыми бэкендами:

- **Адаптер ChromaDB:** работа с основным вектор-хранилищем КРЕПОСТИ
- **Адаптер FAISS:** поддержка альтернативного вектор-индекса для случаев, когда нужна максимальная скорость
- Единый интерфейс запроса независимо от бэкенда

## Гибридный поиск

OCC-RAG работает в связке с гибридным поисковым пайплайном:

1. **Vector Search (BGE-M3):** семантический поиск по эмбеддингам. Находит документы, близкие по смыслу к запросу, даже если они не содержат точных ключевых слов.
2. **BM25:** лексический поиск по ключевым словам. Ловит точные совпадения терминов, которые семантический поиск может пропустить.
3. **Reranker (bge-reranker-base):** переранжирование объединённых результатов. Модель-reranker оценивает пары (запрос, документ) и выдаёт финальный рейтинг релевантности.

Комбинация трёх методов обеспечивает высокое качество извлечения: семантический поиск покрывает перефразировки, BM25 ловит точные термины, а reranker убирает ложные срабатывания.
