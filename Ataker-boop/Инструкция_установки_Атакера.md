# Инструкция установки Атакера (Ataker-Boop)

Пошаговое руководство по установке и запуску атакующего модуля системы КРЕПОСТЬ.
Ataker-Boop — это adversarial-тестер, который проверяет SecurityPipeline на прочность.

> **Важно:** Атакующий работает в ГРЯЗНОЙ ЗОНЕ — физически отделён от защиты.
> В production: Крепость на Mac Studio, Атакер на MacBook Air.
> Атакер НЕ имеет доступа к данным защиты — принцип blind testing.

## 1. Что такое Ataker-Boop

Ataker-Boop — это атакующий модуль для adversarial-тестирования SecurityPipeline Крепости.

Что он делает:

1. **Генерирует атаки** — 70 шаблонов по 15 категориям (injection, jailbreak, encoding bypass и др.)
2. **Мутирует атаки** — 17 стратегий мутаций (base64, homoglyph, zero-width chars, XML injection и др.)
3. **Прогоняет через защиту** — каждая атака проходит через SecurityPipeline
4. **Анализирует результаты** — какие атаки обошли защиту, какие слои слабые
5. **Хранит историю** — Attack Vault (SQLite) хранит все атаки и результаты
6. **Генерирует отчёт** — block rate, latency, слабости по категориям

Цикл работы (из роадмапа Крепости, Фаза 2):

```
Генерация атак → Прогон через Pipeline → Анализ провалов →
  → Запись в Vault → Обновление защиты (через Gate) → Повтор
```

## 8. Первый запуск

### Пример 1: Генерация атак (без SecurityPipeline)

Этот пример показывает, как генерировать и мутировать атаки:

```python
#!/usr/bin/env python3
"""Генерация атак без прогона через Pipeline."""
import sys
sys.path.insert(0, ".")

from ataker import AttackGenerator, MutationEngine, AttackCategory

# Создаём генератор
engine = MutationEngine(seed=42)
gen = AttackGenerator(mutation_engine=engine)

# Генерируем атаки для одной категории
payloads = gen.generate_from_templates(
    categories=[AttackCategory.DIRECT_INJECTION],
    mutations_per_template=2,  # 2 мутации на шаблон
)

print(f"Сгенерировано {len(payloads)} атак\n")

# Показываем первые 5
for p in payloads[:5]:
    print(f"ID: {p.id}")
    print(f"  Оригинал:  {p.original[:80]}")
    print(f"  Мутация:   {p.mutated[:80]}")
    print(f"  Стратегия: {p.mutations_applied}")
    print(f"  Целевой слой: {p.expected_layer}")
    print()
```

Сохраните как `example_generate.py` в папке `Ataker-boop/` и запустите:

```bash
cd ~/Krepost-V3/Ataker-boop
PYTHONPATH=. python3 example_generate.py
```

### Пример 3: Red Team Loop (с SecurityPipeline)

```python
#!/usr/bin/env python3
"""Полный Red Team цикл с реальным SecurityPipeline."""
import asyncio
import sys

sys.path.insert(0, ".")
sys.path.insert(0, "..")

from ataker import RedTeamLoop, AttackVault, AttackCategory
from ataker.http_client import KrepostHTTPClient  # HTTP, не прямой импорт

async def main():
    # Атакер на Air → Крепость на Studio через HTTP
    pipeline = KrepostHTTPClient(base_url="http://<STUDIO_IP>:8000")

    # Создаём Vault для сохранения результатов
    vault = AttackVault(db_path="vault_data/attacks.db")

    # Создаём Red Team Loop
    loop = RedTeamLoop(
        pipeline=pipeline,
        vault=vault,
        seed=42,
    )

    # Запускаем! Ограничим 50 атаками для быстрого теста
    report = await loop.run(
        categories=[
            AttackCategory.DIRECT_INJECTION,
            AttackCategory.ENCODING_BYPASS,
            AttackCategory.HOMOGLYPH,
            AttackCategory.XML_INJECTION,
        ],
        mutations_per_template=2,
        include_chained=True,
        max_attacks=50,
    )

    # Выводим отчёт
    print(report.summary())

    # Статистика Vault
    print(f"\n{'='*60}")
    stats = vault.get_stats()
    print(f"Vault: {stats['total_payloads']} атак, "
          f"{stats['tested']} протестировано, "
          f"{stats['bypassed']} обошли защиту")

    # Показываем слабости
    weaknesses = vault.get_weaknesses()
    if weaknesses:
        print(f"\nОбнаружено {len(weaknesses)} слабостей:")
        for w in weaknesses[:5]:
            print(f"  - [{w['category']}] {w['description'][:80]}")

asyncio.run(main())
```

Сохраните как `example_redteam.py` и запустите:

```bash
cd ~/Krepost-V3/Ataker-boop
PYTHONPATH=.:.. python3 example_redteam.py
```

## 9. Работа с Attack Vault

Attack Vault — это SQLite база данных, которая хранит:
- Все атаки (шаблонные + сгенерированные)
- Результаты тестирования
- Обнаруженные слабости

### Просмотр статистики

```python
from ataker import AttackVault

vault = AttackVault(db_path="vault_data/attacks.db")
```

## 10. Подключение LLM для генерации атак

По умолчанию Атакер использует шаблонные атаки. Для генерации НОВЫХ вариаций можно подключить LLM.

### Через LM Studio (Llama-3.1-8B-abliterated)

```bash
# Загрузите GGUF в LM Studio (см. MODELS.md)
# Запустите LM Studio на Air
# → Load model → Local Server → Start Server
# По умолчанию: http://127.0.0.1:1234
```

```python
import os
from openai import OpenAI
from ataker import AttackGenerator, AttackCategory

# Подключаем Llama-3.1-8B-abliterated через LM Studio
client = OpenAI(
    base_url=os.environ.get("ATAKER_JUDGE_URL", "http://127.0.0.1:1234/v1"),
    api_key="lm-studio",
)

MODEL = os.environ.get("ATAKER_JUDGE_MODEL", "Meta-Llama-3.1-8B-Instruct-abliterated-Q5_K_L")

def llm_generate(prompt: str) -> str:
    response = client.chat.completions.create(
        model=MODEL,
        messages=[{"role": "user", "content": prompt}],
        max_tokens=2000,
    )
    return response.choices[0].message.content

# Создаём генератор с LLM
gen = AttackGenerator(llm_generate=llm_generate)

# Генерируем новые атаки через LLM
new_attacks = gen.generate_with_llm(
    category=AttackCategory.JAILBREAK,
    count=10,
)

print(f"LLM сгенерировал {len(new_attacks)} новых атак")
for a in new_attacks:
    print(f"  - {a.mutated[:80]}...")
```

## 11. Полный Red Team цикл

Вот полный пример production-подобного цикла:

```python
#!/usr/bin/env python3
"""
Полный Red Team цикл с сохранением результатов и отчётом.
"""
import asyncio
import json
import sys
from pathlib import Path

sys.path.insert(0, ".")
sys.path.insert(0, "..")

from ataker import (
    RedTeamLoop, AttackVault, AttackGenerator,
    MutationEngine, AttackCategory,
)
from ataker.http_client import KrepostHTTPClient  # HTTP, не прямой импорт

async def main():
    # === НАСТРОЙКА ===
    # Атакер на Air → Крепость на Studio через HTTP
    pipeline = KrepostHTTPClient(base_url="http://<STUDIO_IP>:8000")
    vault = AttackVault(db_path="vault_data/full_test.db")
    engine = MutationEngine(seed=None)  # случайный seed для разнообразия

    loop = RedTeamLoop(
        pipeline=pipeline,
        vault=vault,
        mutation_engine=engine,
    )

    # === ПРОГОН ===
    print("Запуск полного Red Team цикла...")
    print("Категории: все 15")
    print("Мутации: 3 на шаблон + цепочки глубины 2")
    print()

    report = await loop.run(
        categories=None,           # все 15 категорий
        mutations_per_template=3,  # 3 мутации на каждый шаблон
        include_chained=True,      # + цепочки мутаций
        chain_depth=2,             # глубина цепочки
    )

    # === ОТЧЁТ ===
    print(report.summary())

    # Сохраняем отчёт в JSON
    report_path = Path("vault_data/report.json")
    report_path.write_text(
        json.dumps(report.to_dict(), ensure_ascii=False, indent=2),
        encoding="utf-8",
    )
    print(f"\nОтчёт сохранён: {report_path}")

    # === АНАЛИЗ СЛАБОСТЕЙ ===
    stats = vault.get_stats()
    print(f"\n{'='*60}")
    print(f"ИТОГО:")
    print(f"  Атак в Vault: {stats['total_payloads']}")
    print(f"  Block rate:   {stats['block_rate']:.1%}")
    print(f"  Слабостей:    {stats['open_weaknesses']}")

    if stats['open_weaknesses'] > 0:
        print(f"\nОБНАРУЖЕННЫЕ СЛАБОСТИ:")
        for w in vault.get_weaknesses()[:10]:
            print(f"  [{w['severity'].upper()}] {w['description'][:100]}")

asyncio.run(main())
```

## 12. Мутации — справочник

| # | Мутация | Целевой слой | Что делает |
|---|---------|-------------|------------|
| 1 | `base64` | Layer 1 | Кодирует payload в base64 и просит декодировать |
| 2 | `homoglyph` | Layer 1 | Заменяет 40% латинских букв кириллическими аналогами |
| 3 | `zero_width` | Layer 1 | Вставляет невидимые Unicode-символы между буквами |
| 4 | `case_mix` | Layer 1 | Случайно меняет регистр: iGnOrE pReViOuS |
| 5 | `token_split` | Layer 1 | Разбивает слова дефисами: i-g-n-o-r-e |
| 6 | `rot13` | Layer 1 | ROT13 кодирование с просьбой декодировать |
| 7 | `reverse` | Layer 1 | Переворачивает каждое слово: erongi suoiverp |
| 8 | `leetspeak` | Layer 1 | Замена букв цифрами: 1gn0r3 pr3v10u5 |
| 9 | `whitespace` | Layer 1 | Вставляет лишние пробелы: i g n o r e |
| 10 | `prefix_innocent` | Layer 2 | Добавляет безобидный контекст перед атакой |
| 11 | `suffix_distract` | Layer 2 | Добавляет отвлекающий текст после атаки |
| 12 | `language_switch` | Layer 1-2 | Оборачивает в другой язык (JP, RU, ES, DE) |
| 13 | `markdown_wrap` | Layer 1 | Оборачивает в \`\`\`system блок |
| 14 | `xml_wrap` | Layer 1 | Оборачивает в `<system>`, CDATA, или `<\|im_start\|>` |
| 15 | `json_wrap` | Layer 1 | Оборачивает в `{"role": "system", "content": "..."}` |
| 16 | `multi_encoding` | Layer 1-2 | Комбинация: base64 + homoglyph |

## 13. Категории атак — справочник

| # | Категория | Шаблонов | Целевой слой | Описание |
|---|-----------|----------|-------------|----------|
| 1 | `direct_injection` | 8 | Layer 1 | "Ignore previous instructions" |
| 2 | `role_hijack` | 7 | Layer 2 | DAN, AIM, STAN, developer mode |
| 3 | `system_prompt_leak` | 6 | Layer 2 | Извлечение системного промпта |
| 4 | `encoding_bypass` | 4 | Layer 1 | base64, hex, URL-encoding |
| 5 | `homoglyph` | 4 | Layer 1 | Кириллица вместо латиницы |
| 6 | `chat_template` | 5 | Layer 1 | `<\|im_start\|>`, `[INST]`, `system:` |
| 7 | `xml_injection` | 4 | Layer 1 | CDATA, `<system>`, `<?xml` |
| 8 | `context_overflow` | 2 | Layer 2 | Заполнение контекста + инструкция в конце |
| 9 | `multilingual` | 5 | Layer 1 | Атаки на CN, AR, KR, RU |
| 10 | `social_engineering` | 4 | Layer 2 | Авторитет, срочность, academia |
| 11 | `pii_extraction` | 4 | Layer 4 | Извлечение PII, API keys |
| 12 | `multi_turn` | 3 | Layer 3 | Многоходовые атаки |
| 13 | `jailbreak` | 4 | Layer 2 | Hypothetical, grandmother, creative |
| 14 | `adversarial_suffix` | 2 | Layer 2 | GCG-стиль суффиксы |

**Итого: 70 шаблонов × (1 + N мутаций) = сотни уникальных атак.**

## 14. Частые проблемы

### "ModuleNotFoundError: No module named 'ataker'"

**Причина:** PYTHONPATH не включает папку Ataker-boop.

**Решение:**

```bash
cd ~/Krepost-V3/Ataker-boop
PYTHONPATH=. python3 ваш_скрипт.py
```

Или установите пакет:

```bash
pip install -e .
```

### "ModuleNotFoundError: No module named 'krepost'"

**Причина:** Атакер пытается использовать прямой импорт вместо HTTP.

**Решение:** по умолчанию Атакер общается с Крепостью через HTTP,
не через `import krepost`. Убедитесь, что:

```bash
# Крепость запущена на Studio
curl http://<STUDIO_IP>:8000/health

# В скриптах Атакера используется HTTP-клиент, не прямой импорт
# Пример:
#   from ataker.http_client import KrepostHTTPClient
#   pipeline = KrepostHTTPClient(base_url="http://<STUDIO_IP>:8000")
```

Если нужно локальное тестирование — установите Крепость как пакет:
```bash
source ~/Krepost-V3/.venv/bin/activate
cd ~/Krepost-V3
pip install -e .
```

### "ModuleNotFoundError: No module named 'loguru'"

**Причина:** зависимости Атакера не установлены.

**Решение:**

```bash
cd ~/Krepost-V3/Ataker-boop
pip install -e .
```

### "sqlite3.OperationalError" при работе с Vault

**Причина:** нет директории для базы данных.

**Решение:**

```bash
mkdir -p vault_data
```

### Атакер показывает 100% block rate

**Это нормально**, если вы тестируете только Layer 1 (Regex) без мутаций.
Layer 1 ловит все прямые инъекции из шаблонов.

Чтобы увидеть реальную картину:
- Включите мутации (`mutations_per_template=3`)
- Включите цепочки (`include_chained=True`)
- Добавьте категории, нацеленные на Layer 2+ (jailbreak, social_engineering)

### Атакер показывает 0% block rate

**Причина:** Pipeline создан без Guard-модели, а атаки нацелены на Layer 2.

**Решение:** подключите Guard-модель (см. Инструкцию Крепости, раздел 9.2) или тестируйте только Layer 1:

```python
report = await loop.run(
    categories=[
        AttackCategory.DIRECT_INJECTION,
        AttackCategory.CHAT_TEMPLATE,
        AttackCategory.XML_INJECTION,
    ],
    mutations_per_template=0,
    include_chained=False,
)
```
