# Hopfield Memory Paper

Исследование по Modern Hopfield Memory и её применению в системе КРЕПОСТЬ.

## Modern Hopfield Networks (Ramsauer et al., 2020)

Классические сети Хопфилда (1982) имели фундаментальное ограничение: ёмкость хранения росла линейно с размерностью паттерна (O(d)). Modern Hopfield Networks решают эту проблему, обеспечивая **экспоненциальную ёмкость хранения** — количество паттернов, которые сеть может надёжно хранить и восстанавливать, растёт экспоненциально с размерностью.

## Ключевые свойства

- **Экспоненциальная ёмкость:** в отличие от классических сетей Хопфилда, Modern Hopfield Networks могут хранить экспоненциально больше паттернов без деградации качества извлечения.
- **Content-addressable memory с attention-подобным механизмом:** механизм извлечения паттернов математически эквивалентен attention-слою в трансформерах. Запрос (query) взвешенно агрегирует хранимые паттерны через softmax, что делает систему совместимой с современными архитектурами.
- **Сходимость за один шаг:** в большинстве случаев сеть восстанавливает паттерн за одну итерацию обновления, что критично для low-latency применений.

## Применение в КРЕПОСТИ: Episodic Memory

В системе КРЕПОСТЬ Modern Hopfield Networks используются как **episodic memory** — память эпизодов взаимодействия:

- **Хранение query+response пар:** каждый обработанный запрос и ответ сохраняются как эпизод с двумя эмбеддингами (dual embeddings) — один для запроса, один для ответа.
- **Dual embeddings:** запрос кодируется через BGE-M3 в вектор, ответ кодируется отдельно. При поиске похожих эпизодов система сравнивает оба вектора, что повышает точность извлечения.
- **Exp-decay ranking:** релевантность эпизодов ранжируется с экспоненциальным затуханием по времени. Недавние эпизоды имеют больший вес, старые постепенно теряют влияние, но не удаляются.

## Lifelong Learning без дообучения

Ключевое преимущество подхода — **lifelong learning без дообучения весов модели**. Новые знания добавляются путём записи новых эпизодов в Hopfield Memory, а не через fine-tuning. Это означает:
- Нулевое время на дообучение
- Отсутствие catastrophic forgetting
- Мгновенное добавление новых знаний

## Решение label_conflicts

При конфликтах меток (когда похожие запросы имели разные классификации) система использует **взвешенное голосование** среди извлечённых эпизодов. Вес каждого эпизода определяется комбинацией cosine similarity и exp-decay factor, что обеспечивает корректное разрешение конфликтов в пользу наиболее релевантных и свежих примеров.
