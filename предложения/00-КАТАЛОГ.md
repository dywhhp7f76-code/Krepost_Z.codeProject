## Встроенный механизм гейтинга

Модуль `krepost/governance/` — программный гейт-контроллер, встроенный в систему Крепости.
Принцип: **fail-closed** — если предложение не имеет статуса APPROVED, интеграция заблокирована.

| Файл | Назначение |
|------|------------|
| `krepost/governance/gate.py` | `ImprovementGate` — SQLite-хранилище предложений, audit log, API для approve/reject/revise |
| `krepost/governance/rules.yaml` | Правила гейтинга: обязательное одобрение, полный список изменений, уведомление, аудит |
| `krepost/governance/__init__.py` | Экспорт: `ImprovementGate`, `Proposal`, `ProposalStatus` |

### API гейта

```python
from krepost.governance import ImprovementGate, Proposal

gate = ImprovementGate()

# Агент создаёт предложение
proposal = Proposal(id="01", title="Threat Intel", description="...", category="defense")
gate.submit(proposal)        # → статус: PENDING

# Оператор принимает решение
gate.approve("01", comment="Одобрено, приступать")
gate.reject("02", comment="Слишком рано")
gate.request_revision("03", comment="Доработать риски")

# Проверка перед интеграцией (fail-closed)
if gate.is_approved("01"):
    integrate(proposal)
    gate.mark_integrated("01")

# Аудит всех решений
gate.get_audit_log("01")     # → полная история изменений статуса
```

## Как работает процесс

1. Предложение готовится в этой папке как markdown-файл
2. `ImprovementGate.submit()` регистрирует предложение в SQLite
3. Каталог (этот файл) обновляется
4. Владелец получает уведомление и принимает решение:
   - **✅ ОДОБРЕНО** — `gate.approve(id)` — предложение принято к реализации
   - **❌ ОТКЛОНЕНО** — `gate.reject(id)` — предложение отклонено с причиной
   - **🔄 ДОРАБОТАТЬ** — `gate.request_revision(id)` — требуются изменения
5. `gate.is_approved(id)` — ТОЛЬКО одобренные предложения интегрируются
6. `gate.mark_integrated(id)` — статус обновляется
7. Все решения записываются в `audit_log` — полная трассировка

## Обозначения статусов

| Статус | Значение |
|---|---|
| Ожидает рассмотрения | Предложение подготовлено, ждёт решения владельца |
| ОДОБРЕНО | Принято к реализации |
| ОТКЛОНЕНО | Отклонено владельцем |
| ДОРАБОТАТЬ | Требуются изменения |
| В РАБОТЕ | Идёт реализация |
| ЗАВЕРШЕНО | Интегрировано в основной код |

## Приоритеты

| Приоритет | Критерии |
|---|---|
| **Высокий** | Блокирует другие предложения, критично для безопасности или основной функциональности |
| **Средний** | Важное улучшение, но система работает без него |
| **Низкий** | Оптимизация, удобство, не влияет на ядро |
